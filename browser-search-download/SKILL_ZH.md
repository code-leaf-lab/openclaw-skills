# 浏览器搜索下载技能

## 描述
自动化在Web应用程序（如千川/巨量引擎）中搜索数据并将结果下载到本地存储的过程。该技能处理：
- 导航到搜索页面
- 通过精确的两步日期选择设置搜索参数
- 通过特定按钮识别触发数据导出/下载任务
- 使用重试逻辑监控下载完成情况
- 在本地系统上定位下载的文件

## 使用方法
```
sessions_spawn(
  runtime="subagent",
  agentId="browser-search-download",
  task="从 [平台] 搜索并下载数据，参数：[详细信息]"
)
```

## 配置
无需特殊配置。使用默认浏览器配置文件（"user"）进行已认证会话。

## 工作流程步骤（最新流程）
1. **导航到目标页面** - 在用户的浏览器中打开指定的URL
2. **两步日期选择**：
   - 步骤2a：点击日期范围组件以打开日期选择器
   - 步骤2b：等待日期选择器出现，然后点击预设选项（如"最近30天"）
3. **应用数据维度** - 选择所需的维度（如"抖音号"）
4. **精确的下载按钮识别** - 查找并点击具有 `name="oc-icon-download"` 属性的按钮
5. **等待下载完成** - 等待10秒让文件下载
6. **验证文件存在性** - 检查下载文件夹中是否存在下载的文件
7. **重试机制** - 如果文件不存在，重复整个过程
8. **报告本地文件路径** - 成功时返回确切的本地文件路径

## 所需权限
此技能需要以下权限才能正常工作：
- **浏览器自动化**：完全访问控制用户的浏览器（`profile="user"`）
- **文件系统访问**：读取下载文件夹以验证文件存在性
- **命令执行**：能够运行PowerShell命令进行文件验证
- **网络访问**：访问目标网站（千川、巨量引擎等）

如果文件系统访问受限，该技能将完成浏览器自动化步骤，但无法验证或报告确切的本地文件路径。

## 支持的平台
- 千川（巨量引擎）广告平台
- 具有类似下载任务工作流程的其他平台

## 限制
- 要求用户登录到目标平台
- 假设默认浏览器下载位置（`~/Downloads`）
- 可能需要针对不同UI结构进行平台特定调整
- 文件系统验证可能会被安全策略阻止

## 示例用例
- "下载最近30天按抖音号分类的千川广告表现数据"
- "导出按产品类别过滤的抖音小店销售数据"
- "从巨量引擎平台获取月度广告报告"

## 错误处理
- 使用 `name="oc-icon-download"` 属性实现精确的元素识别
- 包含失败下载的重试逻辑
- 在每个工作流程步骤中提供清晰的状态更新
- 完成时报告本地文件路径或详细的错误信息
